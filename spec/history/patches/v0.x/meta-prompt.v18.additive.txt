D) LABELING RULES — NO-MATCH BEHAVIOR (MANDATORY)
- labeling policy MUST specify:
  - no_match_action: one of [empty_labels, default_label, blocked]
  - confidence_mode: one of [rule_binary, model_probability]
- Default baseline (LABELING_MIN_V1):
  - no_match_action = empty_labels
  - confidence_mode = rule_binary
- Semantics:
  - rule_binary: any matched label is assigned label_confidence=1.0; if no labels produced then label_confidence=null.
  - model_probability: label_confidence is a probability in [0,1] and must be >= label_confidence_threshold to persist labels.
- If no_match_action is missing: BLOCKED: labeling_no_match_undefined.
- If confidence_mode is missing: BLOCKED: labeling_confidence_mode_missing.

LABELING QUALITY CONTROLS (NORMATIVE; DETERMINISTIC)
- If the active labeling policy contains a "label_quality" object, the labeling implementation MUST apply it deterministically.
- Supported keys (all optional; defaults are policy-defined):
  - dedupe (boolean): if true, the same label is emitted at most once per label_type.
  - min_occurrences (integer >= 1): a label is emitted only if total occurrences across evaluated sections >= min_occurrences.
  - max_labels_per_type (integer >= 1): after scoring, keep only the top N labels per label_type.
  - section_weighting (object): weights for ["title","headings","body"] as positive integers. Missing keys default to 1.
  - primary_label_strategy (string): "highest_score_per_type" only.
  - emit_primary_labels_to_extensions (boolean): if true, emit primary labels under chunk.extensions.<extensions_key>.primary_labels.
  - extensions_key (string): default "labeling".

Scoring algorithm (mandatory when label_quality is present)
1) Text segmentation (deterministic):
   - title_text: the document title string (if available; else empty string).
   - headings_text: concatenation of extracted headings (if available; else empty string).
   - body_text: extracted_text.
2) Normalization:
   - For scoring, each section is normalized as: lowercase + collapse whitespace to single spaces.
3) Term matching:
   - For each rule entry (label_type -> list of rules), each rule contains match_any terms and a target label.
   - For each match_any term, count non-overlapping substring occurrences in each section (title/headings/body) after normalization.
   - Section score contribution = occurrences_in_section * section_weighting[section].
   - Label score = sum of contributions across all terms mapped to that label.
4) Thresholding:
   - Raw occurrence filter (mandatory):
- Compute raw_total_occurrences as the unweighted total occurrences across sections.
- If raw_total_occurrences < min_occurrences: do not emit the label (stop evaluating it).
Weighted scoring phase:
- Only labels that pass the raw occurrence filter enter weighted scoring and ranking.
5) Selection:
   - Sort candidate labels by (score desc, label lexicographic asc) and keep up to max_labels_per_type.
6) Primary labels:
   - primary_label_strategy = highest_score_per_type selects the top label after sorting per label_type.
   - If emit_primary_labels_to_extensions=true, output:
     chunk.extensions.<extensions_key>.primary_labels = { "<label_type>": "<label>" }
     chunk.extensions.<extensions_key>.label_scores = { "<label_type>": { "<label>": <score> } } (optional but deterministic if emitted)

Outputs (mandatory)
- labels array MUST include the emitted labels (across all label_types) after quality controls.
- label_confidence behavior MUST follow confidence_mode:
  - rule_binary: if labels non-empty => 1.0; else null.
  - model_probability: probability in [0,1], filtered by label_confidence_threshold.

SCOPE CLASSIFIER (NORMATIVE; DETERMINISTIC)
- The scope gate MUST use a deterministic, auditable classifier for allowlist_taxonomy vs denylist_taxonomy.
- No probabilistic-only or opaque classification is allowed by default. Optional model usage must be explainable and logged, but the baseline MUST be rule-based.

Inputs
- scope_policy.allowlist_taxonomy (list of allowed scope classes)
- scope_policy.denylist_taxonomy (list of denied scope classes)
- scope_policy.scope_confidence_threshold (0..1)
- scope_policy must provide classifier parameters under scope_policy.classifier_v1 (mandatory for v16+).

Deterministic algorithm (classifier_v1)
1) Text segmentation (deterministic):
   - title_text: document title if available else "".
   - headings_text: concatenated headings if available else "".
   - body_text: extracted_text.
2) Normalization:
   - lowercase; collapse whitespace; preserve ASCII letters/digits and standard punctuation.
3) Term dictionaries:
   - allow_terms: mapping {scope_class: [terms]} and deny_terms: mapping {deny_class: [terms]}.
   - Each term is counted as non-overlapping substring occurrences in each section.
4) Scoring:
   - weights: section_weighting = {"title":W_t, "headings":W_h, "body":W_b} (positive integers).
   - allow_score(scope_class) = Σ(occurrences(term,section)*weight(section)) across its terms and sections.
   - deny_score(deny_class) = Σ(occurrences(term,section)*weight(section)) across its terms and sections.
   - allow_total = max allow_score across allow_classes (0 if none).
   - deny_total  = max deny_score across deny_classes (0 if none).
5) Decision (fail-closed):
   - If deny_total >= deny_threshold: OUT_OF_SCOPE (drop) with reason="deny_threshold_met".
   - Else if allow_total >= allow_threshold AND (allow_total - deny_total) >= allow_margin: IN_SCOPE.
   - Else: BLOCKED: scope_gate_low_confidence (do not persist).
6) scope_class selection:
   - scope_class = argmax allow_score; tie-break: (score desc, class lexicographic asc).
7) scope_confidence (0..1):
   - confidence = min(1.0, max(0.0, (allow_total - deny_total) / confidence_divisor))
   - If confidence < scope_confidence_threshold => BLOCKED: scope_gate_low_confidence.
8) Audit outputs (mandatory):
   - chunk.scope_class, chunk.scope_confidence, chunk.scope_policy_fingerprint.
   - chunk.extensions.scope = {
       "allow_top_class": <scope_class>,
       "allow_top_score": <allow_total>,
       "deny_top_class": <deny_class_or_null>,
       "deny_top_score": <deny_total>,
       "decision": "IN_SCOPE"|"OUT_OF_SCOPE"|"BLOCKED",
       "classifier_version": "classifier_v1"
     }

QUALITY SCORE (NORMATIVE; DETERMINISTIC)
- The quality gate MUST compute a deterministic quality_score in [0,1] and quality_flags[].
- Baseline quality scoring MUST be rule-based. Optional model-based quality is out-of-scope unless explicitly enabled by policy and logged.

Inputs (quality.QUALITY_MIN_V1 extended)
- min_quality_score (0..1)
- max_duplicate_ratio (0..1)
- max_missing_metadata_ratio (0..1)
- schema_validation_required (bool)
- quality_policy must provide scoring parameters under quality_policy.scoring_rubric_v1 (mandatory for v16+).

Deterministic scoring rubric (scoring_rubric_v1)
1) Start score = 1.0
2) Mandatory checks (set flags and apply penalties):
   - EMPTY_TEXT: if extracted_text normalized length == 0 => score = 0.0; flags += ["EMPTY_TEXT"] (hard fail).
   - TOO_SHORT: if token_count < min_tokens => score -= p_too_short; flags += ["TOO_SHORT"].
   - MISSING_TITLE: if title missing/null => score -= p_missing_title; flags += ["MISSING_TITLE"].
   - MISSING_SUMMARY: if summary missing/null => score -= p_missing_summary; flags += ["MISSING_SUMMARY"].
   - PARSE_WARNING: if parse_warnings_present => score -= p_parse_warning; flags += ["PARSE_WARNING"].
3) Duplicate ratio (document-level, deterministic):
   - duplicate_ratio = duplicates / max(1,total_chunks) where duplicates are chunks with identical content_hash within doc.
   - If duplicate_ratio > max_duplicate_ratio => score -= p_duplicate_ratio; flags += ["DUPLICATE_RATIO_HIGH"].
4) Missing metadata ratio (document-level, deterministic):
   - missing_metadata_fields_count over [title, summary, content_type, section_path] per chunk.
   - missing_metadata_ratio = missing / max(1, expected_fields_count).
   - If missing_metadata_ratio > max_missing_metadata_ratio => score -= p_missing_metadata_ratio; flags += ["MISSING_METADATA_HIGH"].
5) Clamp score to [0,1].
6) Gate:
   - If score < min_quality_score => BLOCKED: data_quality_gate.
