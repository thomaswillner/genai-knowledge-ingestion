# Dependency Bootstrap (Local First-Run Accelerator)

Goal
- Reduce time-to-first-working-run by providing a local bootstrap path for mandatory dependencies:
  - Apache Tika Server
  - Presidio Analyzer + Presidio Anonymizer
  - Gitleaks (CLI/native or Docker)

What this is
- A developer bootstrap to get through the first end-to-end scaffold + VERIFY run.
- Not a production deployment guide.

## 1) Start Tika + Presidio via Docker Compose
File: infra/docker-compose.dependencies.yml

Command:
- docker compose -f infra/docker-compose.dependencies.yml up -d

Notes
- Tika /status endpoint:
  - If you need /status, run Tika with -status (see compose file comment) and set:
    config/starter-config.json => tika.status_endpoint_policy="required"
  - If you do not run -status, keep:
    tika.status_endpoint_policy="preferred" (fallback probe allowed)


Notes
- Presidio Docker images are pulled from Microsoft Container Registry (MCR) per Presidio installation docs.

## 2) Configure detector endpoints in the repo
Presidio detector config:
- spec/policies/detectors/presidio.PRESIDIO_EU_PII_V1.json
  - analyzer_service_url: http://127.0.0.1:5002
  - anonymizer_service_url: http://127.0.0.1:5001

Gitleaks detector config:
- spec/policies/detectors/gitleaks.GITLEAKS_SECRETS_V1.json
  - exec_mode: native OR docker
  - native: set gitleaks_path to your installed binary path
  - docker: set gitleaks_image to zricethezav/gitleaks:<tag>

## 3) Validate preflight quickly
- Tika: curl http://127.0.0.1:9998/tika (probe)
- Presidio Analyzer: curl http://127.0.0.1:5002/health (if exposed) OR call /analyze endpoint per Presidio docs
- Presidio Anonymizer: curl http://127.0.0.1:5001/health (if exposed) OR call /anonymize endpoint per Presidio docs

## 4) Run the harness
- Use the one-command test harness required by the spec (to be generated by the build agent).
- Validate:
  - schema validation
  - dry-run pipeline
  - gate behavior on fixtures in tests/fixtures/
